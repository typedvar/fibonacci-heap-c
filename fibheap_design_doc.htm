1         Introduction

The software library described in this document provides a generic implementation of the Fibonacci heap for keys with integer and real data types. The implementation allows extending the Fibonacci heap to be able to handle generic data types as key values.

The design and implementation details of the library along with its API are covered in this document. The library is written in the C programming language.

2         Data Structures

Fibonacci heaps are complex constructs and their realization involves a variety of data structures. The design of the library takes into consideration the need to support generic data types as key values in the Fibonacci heap. Subsequently, our implementation supports a generic type, apart from integer and real types for key values. The generic data-type support is accomplished using a pointer to a void. To manage generic types the library accepts a comparator function that should impose a less-than ordering when two instances of the data is supplied to it. Additionally, to perform the decrease-key operation on such non-trivial types the library accepts a subtraction function (i.e. data types other than float or integer). These two functions are supplied by the library user during initialization.

This section describes the various data structures and types used in the library.

2.1       heapkey_t

To be able to design Fibonacci heaps that can support integer, real and generic data types, we introduce the type heapkey_t. This type allows us to store keys of types other than integer or real. A union is employed to create the generic data holder.

typedef union key

{

int intdata; /* integer data */

float realdata; /* real data */

void *genericdata; /* generic data */

} heapkey_t;

2.2       kind_t

kind_t defines an enumeration data type. This is used to identify the data-type of keys used in the Fibonacci heap.

typedef enum kind

{

KND_UNKNOWNN = -1,

KND_INT,

KND_FLOAT,

KND_GENERIC,

MAX_KINDS

} kind_t;

2.3       node_t

Nodes within the Fibonacci heap are represented with the node_t type. This type consists of a structural part and a storage part. The structural part defines how a node is connected to other nodes of the Fibonacci heap and is realized by various pointers. To conserve space, a node has a pointer only to its first child; the children are connected through a circular doubly linked list. Each node has a pointer to its parent and has a count of the number of children it has. The number of children removed from a node since it was first made child of another node is stored in the cut-count field. By limiting the allowable cut-count to unity we can model the Fibonacci heap. By suitably configuring the allowable cut-count, we can use this data type to model other types of heaps. The storage part is defined using a heapkey_t variable.

/* forward declaration */

typedef struct node node_t;

struct node

{

/* structural data */

node_t *parent; /* pointer to the parent node */

node_t *firstchild; /* pointer to the first child */

node_t *leftsibling; /* pointer to the left sibling */

node_t *rightsibling; /* pointer to the right sibling */

size_t numchildren; /* number of children */

char cutcount; /* number of children removed */

 

/* storage data */

heapkey_t key; /* holds the actual data */

};

2.4       fibheap_t

For all operations on the heap, a handle (opaque pointer) to the Fibonacci heap is available to the library users. An opaque pointer minimises impact on library users in the event of changes to the underlying implementation of the Fibonacci heap. Thus in the library header, the type fibheap_t would look like:

typedef struct fibheap* fibheap_t; /* handle to the Fibonacci heap */

 

Subsequently, the Fibonacci heap API (Application Programming Interface) would consist of the following functions:

fibheap_t createheap( fibheap_t *f );

 

void destroyheap( fibheap_t f );

 

void insert( fibheap_t f, node_t* node );

 

node_t *deletemin( fibheap_t f );

 

void decreasekey( fibheap_t f, node_t *node, heapkey_t delta );

 

node_t *findmin( fibheap_t f );

 

void delete( fibheap_t f, node_t *node );

 

fibheap_t merge( fibheap_t firstheap, fibheap_t secondheap, fibheap_t *outheap );

 

We note that a Fibonacci heap will contain homogeneous data (i.e. data of the same kind). We introduce two auxiliary functions in our library that sets and gets the kind of data contained within a Fibonacci heap. Alternatively, we could have modified the createheap function to accept a second parameter of the kind_t type to define the type of Fibonacci heap needed during creation.

void setkind( fibheap_t f, kind_t kind );

 

kind_t getkind( fibheap_t f );

 

Note that though the fibheap_t is made an opaque pointer, the data types heapkey_t, and kind_t are defined in entirety and is accessible to the library user.

2.5       comparator_t

To support generic data types, the library accepts a comparator function. We introduce the type comparator_t, which is a pointer to a function defined as follows:

 

/* comparator_t : pointer to function

* param1 : lhs

* param2 : rhs

* returns : 0, if lhs == rhs, -1 if lhs < rhs, 1 if lhs > rhs

*/

typedef int ( *comparator_t )( heapkey_t lhs, heapkey_t rhs );

 

The comparator function should be defined by the user, and should impose a strict less-than ordering between two keys. For integer and real data types, this function are not required and defaults to the C operator <. We provide auxiliary getter and setter functions for the comparator.

 

void setcomparator( fibheap_t f, comparator_t comparator );

 

comparator_t getcomparator( fibheap_t f, comparator_t comparator );

2.6       subtractor_t

We introduce another type subtractor_t, a pointer to a function, to support generic types in the Fibonacci heap. This type is defined as follows:

 

typedef heapkey_t ( *subtractor_t )( heapkey_t lhs, heapkey_t rhs );

 

The subtractor function accepts two arguments of type heapkey_t and reduces the first argument by an amount specified by the second argument and returns the resulting value. How the reduction is performed is dependent on the actual data contained within the generic type. The subtractor function is not needed for heaps that would contain integer and real types as keys, and defaults to the C operator in such cases. We provide auxiliary getter and setter functions for the subtractor.

 

void setsubtractor( fibheap_t f, subtractor_t subtractor );

 

subtractor_t getsubtractor( fibheap_t f, subtractor_t subtractor );

2.7       fibheap_internal_t

The fibheap_internal_t data type defines the actual heap and is not visible to the library user. This type contains a pointer to a the node in the root-list with the minimum key value, a kind variable to denote the data type of the values stored in the heap nodes, and a numnodes variable to store the total number of nodes in the heap.

typedef struct fibheap

{

node_t *min; /* pointer to the tree root with minimum key */

kind_t kind; /* data type of the node values */

comparator_t comparator; /* function pointer to the comparator */

subtractor_t subtractor; /* function pointer to the subtractor */

unsigned int numnodes; /* number of nodes in the heap */

} fibheap_internal_t;

 

 

3         Heap Operations

In this section we present the pseudo-code for the various heap operations along with their description.

3.1       Heap creation (createheap)

The function createheap creates an empty min-heap that stores integer key values. To change the heap to be able to store real key values, the function setkind should be used with kind as KIND_FLOAT after the heap is created with a call to createheap.

fibheap_t createheap( fibheap_t *f );

 

1.  allocate memory for storing the fibheap_internal_t data

2.  initialize the allocated memory

3.2       Destroying a Heap (destroyheap)

The function destroyheap deallocates memory associates with a heap and terminates the heap structure.

void destroyheap( fibheap_t f );

 

1.  for each node in the heap, deallocate memory

2.  deallocate memory for the heap

3.3       Inserting a node into the heap (insert)

The function insert adds a node to the heap. This function takes a handle to a heap f, and a pointer to a node_t node that is to be added to the heap.

 

void insert( fibheap_t f, node_t* node );

 

1.  insert node into root-list; the node should have been initialized during creation

2.  make this node the new heap-min if the value contained in it is smaller than the present heap-min

3.  increment the number of nodes stored in the heap

3.4       Deleting the heap-min (deletemin)

The function deletemin deletes the minimum element from the heap and returns a pointer to the deleted node.

 

node_t *deletemin( fibheap_t f );

 

1.  remove the heap-min from the root list

2.  using the pointer to the first child node of the removed heap-min, traverse all children and add the children to the root list

3.  make the rightsibling of the removed heap-min the new heap-min if a rightsibling exists

4.  combine the nodes in the root list which have the same number of children until we are left with nodes such that no two nodes have the same number of children an auxiliary array is used for efficiency.

5.  while performing the above combine-step update the heap-min if a node is found to have a smaller value than current heap-min

 

3.5       Decreasing the key of a node in the heap (decreasekey)

The function decreasekey reduces the value contained in a given node by a supplied delta.

 

void decreasekey( fibheap_t f, node_t *node, heapkey_t delta );

 

1.  check for error conditions associated with subtracting the delta from the value within the given node; i.e. subtraction should not result in negative node value

2.  if the resulting value within the node is less than that of its parent:

a.  remove node from its parents child-list and add it to the root list of f

b.  perform a cascading-cut on its parent based on the cutcount.

c.  make the removed node the new heap-min if the value contained in it is smaller than the present heap-min

 

cascading-cut is performed recursively until condition 2 no longer holds true

3.6       Finding the minimum node in the heap (findmin)

The function findmin find the new heap-min. This straightforward method returns a pointer to the current heap-min

 

node_t *findmin( fibheap_t f );

 

1.  return a pointer to the current heap-min

3.7       Deleting a node from the heap (delete)

The function delete removes a node from the heap. To accomplish this it uses the functions decreasekey and deletemin.

 

void delete( fibheap_t f, node_t *p_node );

 

1.  decrease the value within the node by a value such that it becomes the new heap-min. This can be achieved by examining the current heap-min and appropriately choosing the delta for the decreasekey operation

2.  perform a deletemin on the resulting heap

3.8       Merging two heaps (merge)

The function merge unites two heaps f, and g and returns a newly created heap whose heap-min is the minimum of the heap-min of f and g.

 

fibheap_t merge( fibheap_t f, fibheap_t g, fibheap_t *h );

1.  create a new heap h with a call to createheap

2.  remove all elements from the root list of f and g and add them to the root list of h

3.  assign the heap-min of h to be the minimum of the heap min of both f and g

4.  destroy heaps f and g

 

4         Accessing Input from Files

To be able to load test data from a file, we provide an auxiliary set of functions that allows users the ability to configure the how each entry is interpreted and encapsulated as a generic kind. Each entry forms a record of a generic data type, for whose keys, the user defines the comparator and the subtractor function. The generic data pointer in heapkey_t can be used as a pointer to a record that is loaded from the data file.

5         Hash Tables

We note that since the heap API uses the node_t type, the user should be able access a pointer to the node given a particular key. This calls for a hash-table that can look up a node pointer when supplied with a key. Subsequently the library provides helper functions for hash-table creation, population, access and destruction.

5.1       Hash Table Operations

The hash table operations defined in this section allows the heap library user to look-up node pointers in the heap given a key. We will not describe the data structures associated with the hash table, instead we will use the opaque pointer hash_t to denote a context to the hash table. This context would be required for all subsequent operations on the hash table.

5.1.1       Creating a Hash Table (createht)

This function creates a hash table and returns a handle to it.

hashtable_t createht();

5.1.2       Destroys a Hash Table (deleteht)

This function destroys a hash table and deallocates memory associated with it.

void deleteht( hashtable_t ht );

5.1.3       Adding a Key/Value to the Hash Table (installkv)

This function adds a Key and a Value to the hash table.

void installkv( hashtable_t ht, heapkey_t k, node_t *node );

5.1.4       Looking Up a Key/Value from the Hash Table (lookupkv)

Returns a pointer to the node stored in the hash table against a key k.

node_t *lookupkv( hashtable_t ht, heapkey_t k );

5.1.5       Remove a Key/Value from the Hash Table (removekv)

Removes a key/value pair from the hash table.

void removekv( hashtable_t ht, heapkey_t k );

 

6         Sample User Program

In this section we present a sample program that illustrates how the heap library can be used for string data keys. We omit the details associated with building the program for the sake of brevity. This program illustrates the use of a comparator but does not demonstrate the use of the hash table.

/*

* author : avinandan sengupta

* reg no : 2010CSY7543

* course : CSL630

* assignment : Fibonacci heap

* file : Heap driver for string keys

*/

#include <stdlib.h>

#include <stdio.h>

#include <string.h>

#include "fibheap.h"

 

int compare_strings( heapkey_t lhs, heapkey_t rhs )

{

char* lhs_str = ( char* )lhs.genericdata;

char* rhs_str = ( char* )rhs.genericdata;

return strcmp( lhs_str, rhs_str );

}

 

void test_str_heap()

{

unsigned int i = 0;

node_t *node = NULL;

heapkey_t data;

fibheap_t heap = NULL;

char* test_strings[] = {

"virgo"

, "scorpio"

, "aries"

, "taurus"

, "leo"

, NULL

};

 

heap = createheap( &heap );

setkind( heap, KIND_GENERIC );

setcomparator( heap, &compare_strings );

for( i = 0; test_strings[i]; ++i )

{

data.genericdata = test_strings[i];

node = createnode( data );

insert( heap, node );

}

printf( "Min element is %s \n", ( char* )( findmin( heap )->key.genericdata ) );

destroyheap( heap );

}

 

int main( int argc, char* argv[] )

{

test_str_heap();

return 0;

}
